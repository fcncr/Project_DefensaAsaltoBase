## Avance integrante 2 - Parte jugable

Se implementó la base jugable del proyecto Defensa y Asalto de Base en Python con Tkinter.

Funcionalidades completadas:

- Clases Faccion, Torre, Unidad, Muro y BaseCentral.
- Facciones Medieval, Futurista y Oscura.
- Validación para impedir que defensor y atacante usen la misma facción.
- Catálogo de 3 torres: Básica, Pesada y Mágica.
- Catálogo de 3 unidades: Soldado, Tanque y Unidad Rápida.
- Mapa 10x10 con base central fija en la posición (5,5).
- Interfaz Tkinter para visualizar el mapa.
- Fase defensor para colocar muros y torres.
- Fase atacante para colocar unidades.
- Sistema de dinero para defensor y atacante.
- Botón para ejecutar combate.
- Lógica básica de combate: torres atacan unidades, unidades avanzan hacia la base, atacan muros/torres/base y se determina ganador.
- Sistema de rondas con marcador hasta que un jugador gana 3 rondas.
- Botón de siguiente ronda.
- Modo vender/quitar para corregir objetos colocados antes del combate.
- Pantalla previa para seleccionar facción del defensor y del atacante.

Último avance realizado:

Se agregó la pantalla de selección de facciones antes de abrir el mapa. La pantalla permite elegir facción para defensor y atacante. Si ambos escogen la misma facción, el sistema muestra error. Si las facciones son diferentes, se reinicia el estado de la partida y se abre el mapa con las facciones seleccionadas.

Integrar esta parte jugable con el login, registro, ranking y actualización de victorias.





## Avance integrante 1 - Sistema de usuarios

Se implementó el sistema de usuarios del proyecto Defensa y Asalto de Base en Python con Tkinter y almacenamiento en archivo JSON.

Funcionalidades completadas:

* Creación automática de la carpeta datos si no existe.
* Creación automática del archivo usuarios.json.
* Clase Jugador con usuario, contraseña, victorias como defensor y victorias como atacante.
* Funciones para cargar la información de usuarios desde el archivo JSON.
* Funciones para guardar la información de usuarios en el archivo JSON.
* Función para registrar nuevos jugadores.
* Validación para evitar registrar usuarios repetidos.
* Función para validar inicio de sesión con usuario y contraseña.
* Ventana inicial con opciones de registro, login y ranking.
* Ventana de registro de nuevos jugadores.
* Ventana de login para iniciar sesión con dos jugadores diferentes.
* Validación para impedir que ambos jugadores sean el mismo usuario.
* Ranking de jugadores con top 5 como defensor.
* Ranking de jugadores con top 5 como atacante.
* Actualización automática de victorias al finalizar una partida.
* Cambio del arranque del programa para que el juego inicie desde el sistema de usuarios.

Último avance realizado:

Se conectó el inicio del programa con el sistema de usuarios. Ahora el juego no inicia directamente en la parte jugable, sino que primero muestra una ventana principal donde los jugadores pueden registrarse, iniciar sesión o consultar el ranking. También se implementó el login para dos jugadores diferentes y la actualización de victorias según el rol del jugador ganador.
